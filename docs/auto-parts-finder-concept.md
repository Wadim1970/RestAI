# PartsPilot — AI-подбор запчастей для машин старше 5 лет

> **Позиционирование:** «AI-механик в кармане». Приложение, которое по фото, голосу или VIN
> находит нужную запчасть, подбирает аналоги по цене/качеству и предсказывает, что сломается следующим.

---

## 1. Почему именно этот сегмент

Владелец машины 5+ лет — это особый пользователь:

- **Гарантия кончилась** → обслуживается не у дилера, а в гараже/у частника/сам.
- **Оригинал дорог** → главный вопрос всегда: «какой аналог не развалится через месяц?»
- **Каталоги — ад**: OEM-номера, кроссы, рестайлинги, «подходит/не подходит» — даже опытные путаются.
- **Рынок б/у деталей огромен** (разборки), но фрагментирован и без доверия.
- Часто **не знает, что именно сломалось** — знает только симптом («стучит спереди справа на кочках»).

Существующие игроки (Exist, Emex, Autodoc, Avito-разборки) решают задачу «купить деталь по номеру».
**Никто не решает задачу «у меня что-то стучит — что купить?»** Это и есть ниша.

## 2. WOW-фичи (ядро продукта)

### 2.1 📸 Snap2Part — «Сфоткай — найдём»
Фотографируешь деталь (даже грязную, даже сломанную) → мультимодальный LLM + CV определяет:
что это, для какого узла, вероятный OEM-номер → сразу список аналогов с ценами.
Как Google Lens, но для запчастей. Это фича, которую показывают друзьям в гараже.

### 2.2 🎙️ AI-диагност — «Опиши поломку голосом»
Голосовой ввод (руки в масле — это важно): «на холодную скрипит ремень, потом проходит».
LLM задаёт 2–3 уточняющих вопроса и выдаёт ранжированный список вероятных причин
**с учётом типовых болячек конкретной модели на конкретном пробеге** (RAG по базе
форумов, отзывов, ТСБ и отзывных кампаний). К каждой причине — сразу корзина деталей.

### 2.3 🚗 Цифровой гараж (digital twin автомобиля)
VIN сканируется камерой (у вас уже есть опыт с `@zxing`) → полная комплектация расшифрована.
Дальше все подборы идут без вопросов «а какой у вас двигатель?». Хранится история замен и пробег.

### 2.4 🔮 Предиктивное ТО — «Ваш термостат доживает последние 10 тыс. км»
Статистика отказов по модели × пробег × регион → пуш заранее: «на вашем пробеге у этой модели
обычно умирает помпа, сейчас она стоит 3 200 ₽, через месяц сезонный рост цен». Удержание + повторные покупки.

### 2.5 ⚖️ Умные кроссы: аналоги с «уровнем доверия»
Не просто список из 40 брендов, а 3–4 тира: *Оригинал / OEM-поставщик конвейера / крепкий аналог / бюджет*,
с агрегированным рейтингом «сколько реально ходит» из отзывов владельцев именно этой модели.
Формат ответа: **«Механик советует»** — одна рекомендация + альтернативы, а не стена из позиций.

### 2.6 ♻️ Б/у с разборок + AI-оценка состояния
Агрегация предложений разборок; продавец грузит фото — AI выставляет грейд состояния (A/B/C),
подсвечивает коррозию/трещины. Для машин 10+ лет б/у деталь — часто единственный разумный вариант.
Это самый недооценённый рынок и сильнейший дифференциатор.

### 2.7 📉 «Купить сейчас или подождать»
История цен по позиции (вы уже умеете парсить — puppeteer/cheerio) → график + прогноз + алерт на снижение.
Как Aviasales, только для запчастей.

### 2.8 🥽 AR под капотом (фаза 3)
Навёл камеру под капот → подписи узлов, подсветка «где находится то, что вы ищете»,
пошаговая AR-инструкция замены с таймингом и списком инструмента. Чистый WOW для демо и прессы.

## 3. UX-принципы

- **Chat-first, но не chat-only**: диалог с AI — точка входа, ответы — структурированные карточки
  (деталь, цена, срок, кнопка «в корзину»), а не простыня текста.
- **Один экран — одно действие**: гигантская кнопка «📸 Найти по фото», «🎙️ Описать проблему», «🔢 VIN/номер».
- **Голос везде**: в гараже руки заняты и грязные.
- **Гараж-эстетика без клише**: тёмная тема по умолчанию (в гараже темно), крупная типографика,
  бенто-сетка карточек, микроанимации статусов («ищем на 12 складах…» с живым прогрессом — это продаёт магию).
- **Нулевой порог входа**: первый подбор — без регистрации; аккаунт предлагается в момент,
  когда появилась ценность («сохранить машину в гараж?»).

## 4. Архитектура (эволюция вашего текущего стека)

### MVP (2–3 месяца) — максимально знакомые технологии
```
React 19 + Vite (PWA)  ←→  Supabase (Postgres, Auth, Storage, Edge Functions)
                                 │
                                 ├── pgvector — RAG по базе болячек/мануалов
                                 ├── Edge Functions → Claude API (мультимодальный: фото + текст + диагностика)
                                 └── Парсер-воркеры (puppeteer/cheerio) → цены поставщиков
```

- **PWA, не нативка**: камера, пуши и офлайн доступны, релизы без сторов. Нативная оболочка (Capacitor) — позже.
- **VIN-декодинг**: Laximo / ELCATS API (СНГ) или NHTSA vpic (бесплатно, для старта).
- **Каталог кроссов**: TecDoc-лицензия дорогая — на старте партнёрские API поставщиков (Autodoc, Emex
  отдают кроссы в API) + собственная накопительная база кроссов в Postgres.
- **Snap2Part v1**: без своей CV-модели — фото напрямую в Claude (vision) с промптом-классификатором
  и последующим поиском по каталогу. Своя дообученная модель — только когда будут данные.
- **AI-диагност**: Claude + RAG (pgvector) по корпусу: типовые неисправности, регламенты ТО, отзывные,
  синтез из форумов. Каждый диалог размечается («помогло/не помогло») — это будущий moat из данных.

### Фаза 2 (масштабирование)
- Выделенный поисковый движок (Meilisearch/Typesense) для каталога — Postgres full-text перестанет хватать.
- Очередь (Supabase Queues / Inngest) для парсинга цен и пушей предиктивного ТО.
- Кэш цен с TTL, «горячие» позиции обновляются чаще.
- Кабинет продавца-разборки (по образцу вашего Admin-restai: React + TS + Vite).

### Фаза 3
- Собственная CV-модель распознавания деталей (датасет уже накоплен из Snap2Part).
- AR-модуль (WebXR в PWA либо нативный модуль).
- B2B API для СТО: «подбор корзины по заказ-наряду».

## 5. Монетизация

| Модель | Механика | Когда |
|---|---|---|
| Комиссия / CPA | % с заказа у поставщика-партнёра | MVP |
| Подписка Pro (~299 ₽/мес) | безлимит AI-диагноста, ценовые алерты, предиктивное ТО | Фаза 2 |
| Тариф для разборок | размещение + AI-оценка состояния + буст в выдаче | Фаза 2 |
| B2B для СТО | API подбора + кабинет | Фаза 3 |

Юнит-экономика сегмента приятная: владелец машины 8–12 лет покупает запчасти 4–8 раз в год,
средний чек 3–8 тыс. ₽ — LTV высокий при удержании через «гараж» и предиктивное ТО.

## 6. Roadmap

**MVP (8–12 недель):** VIN-гараж → поиск по номеру/названию → кроссы с тирами качества →
агрегация цен 2–3 поставщиков → AI-диагност v1 (текст+голос) → Snap2Part v1 (через Claude vision).

**V2:** предиктивное ТО, ценовые графики и алерты, б/у с разборок + AI-грейдинг, кабинет продавца.

**V3:** AR под капотом, своя CV-модель, B2B API, соцслой («эти колодки прошли у меня 40 тыс.»).

## 7. Риски и как их снять

- **Точность подбора** — цена ошибки: возврат детали. Решение: всегда показывать «уверенность» подбора,
  при низкой — уточняющий вопрос или проверка по VIN; человеческая валидация спорных кроссов.
- **Доступ к каталогам** — TecDoc дорог. Решение: старт на API поставщиков + накопление собственной базы
  кроссов как актива.
- **Галлюцинации LLM** — диагноз только как «вероятные причины», не «приговор»; RAG с цитированием источника;
  дисклеймер + кнопка «спросить живого механика» (партнёрские СТО — ещё и канал монетизации).
- **Холодный старт отзывов** — синтез из открытых источников на старте, потом UGC за бонусы.

## 8. Что делает продукт запоминаемым (итог)

1. **«Сфоткай — найдём»** — момент магии, который показывают друзьям.
2. **Голосовой AI-диагност**, знающий болячки именно твоей модели.
3. **Предсказание поломок** до того, как они случились.
4. **Б/у с AI-оценкой состояния** — доверие там, где его никогда не было.
5. Всё это — поверх скучной, но идеально сделанной базы: точный VIN-подбор и честное сравнение цен.

WOW приводит пользователя, база (точность + цены) его оставляет.
